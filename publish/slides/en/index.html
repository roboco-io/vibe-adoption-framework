Why now

The tools are already here.
What makes the difference is a system of adoption.

Companies that integrate agents into their organizational system

The effect is like hiring a large number of senior-level developers at low cost. Lead time, quality, and cost structure all change.

Companies stuck at individual experimentation

They use the tools but never convert them into organizational results. It ends with the conclusion "we used AI and nothing changed."

Coding agents already handle much of the work skilled developers used to do. The gap keeps widening.

Core premise

Vibe coding is software development

What changes is not the principles but the economics.

  • 01

    Clear requirements, good architecture, review, testing, documentation — the principles that made human development succeed make agent development succeed too.

  • 02

    What used to be expensive — meticulous reviews, thorough testing, full legacy analysis — agents now perform continuously at low cost.

  • 03

    The question is not "what do we change for AI" but "how do we use agents to realize what we should have been doing all along?"

Foundational Principles

Three principles that cut across every perspective

Principle 1

Agents as workforce leverage

Give agents what you give a capable new hire — access, system explanations, rules, clear instructions, review. Failing organizations blame the agent; successful ones build an onboarding system.

Principle 2

Humans own it

"The agent wrote it, so I don't know" is not allowed. Even when the author changes, the duty to explain does not move. You must be able to answer whenever anyone asks.

Principle 3

Cognitive debt is something to manage

The gap between human understanding and the actual system = cognitive debt. Like technical debt, it is a trade-off to manage, not something to eliminate. The organization decides where to keep deep understanding.

Named Concepts

Two named concepts in VAF

Harness Engineering

The engineering of systematizing the reviews and guardrails humans used to perform so that AI carries them out — a combination of rule files, automated review, hooks, and verification pipelines. A well-built harness elevates human review to a higher level of abstraction: intent, design, trade-offs.

Taste vs. non-taste

Non-taste (security, guardrails, coding conventions) is managed with the repository and enforced on everyone. Only taste (e.g. level of documentation detail) is left to very limited discretion. Rules embedded in the repository become an environment agents cannot violate.

Perspectives in one line

The six questions the perspectives ask

  • P1

    Why adopt, and what do we gain? — When this perspective is weak, the other five dissolve into individual heroics.

  • P2

    What is the human role? — Human value lies not in what you wrote but in what you are accountable for.

  • P3

    What do we enforce? — Agents have no will to break rules. If a rule is not in the environment, they simply don't know it exists.

  • P4

    Can agents work? — You cannot expect results from a new hire with no account.

  • P5

    Do agents know? — Agent performance is proportional to the quality of accessible knowledge.

  • P6

    Who guards quality? — In an organization without a harness, agent speed is defect speed.

J3PilotHarness build · development transition · retrospective · playbook
→
J4TransformOrg-wide rollout · institutionalization · metric tracking · next cycle

VAF puts environment preparation before assessment —
because the agents themselves perform the assessment, analysis, and documentation.

Journey Principles

Four principles for running the journey

  • 01

    Finish small — Going narrow to the finish beats spreading wide and stalling. The experience of completing a cycle is the fuel for expansion.

  • 02

    Don't skip stages — Assessment without preparation runs at human speed; a pilot without assessment produces context-free code; transformation without a pilot is unverified coercion.

  • 03

    Verify with in-depth interviews at every stage — Moving to the next stage while documents diverge from reality is the most expensive failure.

  • 04

    Refresh metrics every cycle — Executive trust comes from metric continuity.

Anti-patterns

Common failures that sink adoption

ANTI-PATTERN

Tool rollout without purpose

Six months later, momentum dies with a "usage is low" report.

ANTI-PATTERN

Starting without a baseline

Unable to prove results, the effort founders at budget renewal.

ANTI-PATTERN

Security review at the end

The answer that comes back is a total block. Bring security in as a design participant.

ANTI-PATTERN

"Ban the agents" after an incident

The cause is not the agent but a gap in the harness.
